## Layer 05 verdict

**Autonomous vs verdict-functions-named-"agent" — definitive: they are verdict functions named "agent."** Every "agent" in the codebase is a label, comment, diagram node, or a `layer:"agent"` string field on a verdict object. The "agents" are three pure functions — `evaluateCoverage` (`governance.js` L266), `evaluateAttribution` (L332), `evaluateForecast` (L396) — each takes `(model, inputs)`, returns a static verdict array, and is called once synchronously inside `evaluateSpine` (L438–446), which `SpinePage.jsx` L45 runs in a `useMemo` on render. Nothing schedules, polls, acts, writes, or routes. The only `setInterval` in `src/` drives a cosmetic pulse animation (`ArchitectureDiagram.jsx` L134). The diagram labels the ring "Autonomous Decision" (`ArchitectureDiagram.jsx` L35), and the doctrine's **Orchestration Agent has no code analog** — there is no "Orchestration" anything in `src/`. The code itself encodes the inconsistency: P&L and Stage are tagged `layer:"governance"` while Coverage/Attribution/Forecast are tagged `layer:"agent"` (`governance.js` L35/278), yet all five are the same kind of synchronous evaluator. The "agent" label is pure taxonomy mapping a governance domain to an outer diagram ring — it maps to no acting code.

**Provenance logging — confirmed ABSENT.** Searched `src/` and `netlify/` for `provenance` → **zero matches**. No decision→inputs→weights→output records. Verdicts carry `metrics:{current,threshold,target}` and prose `diagnosis`/`recommendation` (`governance.js` L41/55), but nothing logs the inputs/weights/model-version that produced a verdict, and verdicts are recomputed-and-discarded each render (the only persisted store is raw actuals in localStorage, `actuals.js` L10/72). Matches the roadmap's own flag: "Provenance logging … the largest unbuilt claim" (`ROADMAP.md` L307).

**Dual-probability (P_base × P_accel) — ABSENT.** Searched `src/` for `p_base`/`p_accel`/`pbase`/`paccel`/`dual.prob` → **zero matches**. Model is single-probability: deal volume derives solely from `sqoToWonRate` (`engine.js` L36/238/264). Acceleration is a flat additive win-rate lift (`accelWinRateLift: 5` pts, `engine.js` L170) + a days-reduced figure — not a P_base × P_accel product, and there is no probability-quadrant motion routing. The dual-probability model exists only in `SPINE-SPEC.md`/`ROADMAP.md` (`ROADMAP.md` L317, flagged 🔴 not in product).
