## Highest-value findings (overclaims to surface)

- **Dual-axis cost model (#3) — PARTIAL.** Both axes are fixed *percentages* of revenue-derived `salesOpex` (`engine.js` L962–966), so they scale identically with revenue. Real contribution-margin number, but fixed costs don't hold flat as volume grows. Only true dollar-floor behavior is the separate headcount-floor logic (L617–661).
- **Coverage-ratio enforcement (#4) — PARTIAL.** `coverageHealth` (L283–284) thresholds the input against itself. Real coverage logic is in governance and only *flags*; it never constrains the target. "Enforcement" overstates "surfacing."
- **Horizon Planner attainment (#10) — latent inconsistency.** Hardcoded `0.85` (L1232) vs the user input `realisticAeAttainment` (=75) the AE Hiring Plan obeys (L482); the two can disagree for identical inputs.
- **AE Quota Benchmarks (#8) — constant, not computed** (honest reference).
- **Phase 0 (#11) — real but equal-weighted**, not the "weighted" model implied by Field-Audit copy.
- **Phase-shifted funnel (#2) — REAL but scoped:** not applied to the monthly ARR build or P&L. If marketing claims "Q4 spend → Q1 pipeline → Q3 close" drives the headline ARR trajectory, that's an overclaim — it drives the coverage/CRO funnel shapes only.
